## 既に存在するコントラクトを使って subgraph 用のプロダクトを作る場合

```bash
graph init \
    --product subgraph-studio
    --from-contract 0xa3fb6BDaEd02F22c5488e744a471C18784393881 \
    --network 43113 \
    --abi ./packages/frontend/src/contracts/MyToken.json \
    subgraph
```

### デプロイに失敗した時の詳しいログの確認方法

[https://graphiql-online.com/](https://graphiql-online.com/)でエンドポイントを[https://api.thegraph.com/index-node/graphql](https://api.thegraph.com/index-node/graphql)に指定して次のクエリを実行する。

```gql
{
  indexingStatuses(
    subgraphs: ["QmbLZDXtK8rBi2HSFqAskqo1E3Fnd7EyywAv3ZZtd4MbHu"]
  ) {
    node
    synced
    health
    fatalError {
      message
      block {
        number
        hash
      }
      handler
    }
    nonFatalErrors {
      message
      block {
        number
        hash
      }
      handler
    }
    chains {
      network
      chainHeadBlock {
        number
      }
      earliestBlock {
        number
      }
      latestBlock {
        number
      }
      lastHealthyBlock {
        number
      }
    }
    entityCount
  }
}
```
